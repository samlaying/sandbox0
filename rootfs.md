# RootFS 架构设计

## 概述

本文档描述 sandbox0 如何使用 SandboxVolume 作为容器的根文件系统（RootFS），实现零拷贝的容器快照和恢复。

## 核心思想

**将容器的整个根文件系统直接挂载到 SandboxVolume，所有写操作自动进入 JuiceFS，实现秒级快照。**

```
传统容器架构:
容器根目录 → overlayfs → 宿主机目录 → 容器删除丢失

sandbox0 架构:
容器根目录 → SandboxVolume (JuiceFS) → 持久化存储 + 秒级快照
```

## 整体架构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           sandbox0 RootFS 架构                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                         沙箱容器                                     │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │                                                                     │   │
│  │  / (根目录)                                                          │   │
│  │  └── 直接挂载到 SandboxVolume                                        │   │
│  │      ├── /workspace   ← 用户工作目录                                  │   │
│  │      ├── /tmp        ← 临时文件                                      │   │
│  │      ├── /home       ← 用户数据                                      │   │
│  │      ├── /var        ← 应用状态                                      │   │
│  │      ├── /etc        ← 配置修改                                      │   │
│  │      └── ... 任何写操作                                              │   │
│  │                                                                      │   │
│  │      └─────────────────────────────────────────────────────────┐    │   │
│  │                                                                 │    │   │
│  │                    全部自动进入 JuiceFS                          │    │   │
│  │                                                                 │    │   │
│  │                    S3 存储 + PostgreSQL 元数据                    │    │   │
│  └─────────────────────────────────────────────────────────────────┴────┘   │
│                                                                             │
│  快照流程 (~600ms):                                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 1. Pause 容器进程 (freeze)                                         │   │
│  │ 2. CRIU dump 内存 (~500ms)                                         │   │
│  │ 3. JuiceFS 元数据快照 (~50ms)  ← 文件系统无需操作！                │   │
│  │ 4. Resume 容器进程                                                 │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  恢复流程 (~300-500ms):                                                     │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 1. 创建新容器                                                         │   │
│  │ 2. 挂载同一个 SandboxVolume  ← 文件系统已经"在那里"了！             │   │
│  │ 3. CRIU restore 内存                                                │   │
│  │ 4. 完成！                                                             │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 工作流程

### 1. SandboxTemplate 创建阶段

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    创建 SandboxTemplate                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  输入:                                                                      │
│  - 镜像: sandbox0/python:3.11                                               │
│  - 配置: sandbox_rootvolume: "template-python-3.11-rootfs"                  │
│                                                                             │
│  处理流程:                                                                  │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ Step 1: 拉取镜像                                                     │   │
│  │ └─ containerd/crun 拉取镜像层到本地                                  │   │
│  │                                                                      │   │
│  │ Step 2: 解压 RootFS                                                  │   │
│  │ └─ 将镜像所有层合并到临时目录                                        │   │
│  │                                                                      │   │
│  │ Step 3: 创建临时 SandboxVolume                                       │   │
│  │ └─ volume-id: "temp-template-python-3.11-rootfs-xxxxx"               │   │
│  │                                                                      │   │
│  │ Step 4: 上传 RootFS 到临时 Volume                                    │   │
│  │ └─ 将解压的 RootFS 复制到 Volume 的挂载点                            │   │
│  │                                                                      │   │
│  │ Step 5: 克隆 Volume 到目标                                           │   │
│  │ └─ juicefs clone temp-volume → template-python-3.11-rootfs           │   │
│  │                                                                      │   │
│  │ Step 6: 清理临时 Volume                                              │   │
│  │ └─ 删除临时 volume                                                   │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  结果:                                                                      │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ SandboxVolume "template-python-3.11-rootfs"                         │   │
│  │ └── 包含完整的镜像 RootFS                                            │   │
│  │     ├── /usr/bin/python3                                             │   │
│  │     ├── /usr/local/lib/python3.11/                                   │   │
│  │     ├── /workspace/  (空目录，等待用户使用)                          │   │
│  │     └── ...                                                          │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  SandboxTemplate CRD:                                                      │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ apiVersion: sandbox0.ai/v1                                          │   │
│  │ kind: SandboxTemplate                                               │   │
│  │ metadata:                                                            │   │
│  │   name: python-3.11                                                 │   │
│  │ spec:                                                                │   │
│  │   image: sandbox0/python:3.11                                       │   │
│  │   sandboxRootVolume: template-python-3.11-rootfs                    │   │
│  │   minIdle: 5                                                        │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3. 容器快照阶段

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    创建容器快照                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  用户请求:                                                                  │
│  POST /api/snapshots                                                        │
│  { "sandbox_id": "sbx-abc123" }                                             │
│                                                                             │
│  处理流程:                                                                  │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ Step 1: Pause 进程                                                  │   │
│  │ └─ 使用 cgroup freezer 冻结所有进程                                  │   │
│  │     echo FROZEN > /sys/fs/cgroup/freezer/sbx-abc123/freezer.state    │   │
│  │     (~10ms)                                                          │   │
│  │                                                                      │   │
│  │ Step 2: CRIU Checkpoint 内存                                         │   │
│  │ └─ criu dump -t <pid> -D /snapshots/sbx-abc123/criu                 │   │
│  │     --shell-job --leave-running                                      │   │
│  │     (~500ms)                                                         │   │
│  │     输出: 内存页、寄存器状态、文件描述符等                            │   │
│  │                                                                      │   │
│  │ Step 3: JuiceFS 快照 (可选)                                         │   │
│  │ └─ juicefs snapshot create sbx-abc123-rootfs                         │   │
│  │     (~50ms)  ← 元数据操作，非常快！                                  │   │
│  │     注意: 文件系统已经持久化了，这个快照只是为了                    │   │
│  │           确保时间点一致性                                          │   │
│  │                                                                      │   │
│  │ Step 4: Resume 进程                                                  │   │
│  │ └─ echo THAWED > /sys/fs/cgroup/freezer/sbx-abc123/freezer.state    │   │
│  │     (~10ms)                                                          │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  总耗时: ~600ms                                                             │
│                                                                             │
│  快照数据:                                                                  │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ Snapshot ID: snap-xyz789                                            │   │
│  │                                                                      │   │
│  │ /snapshots/snap-xyz789/                                             │   │
│  │ ├── criu/           ← CRIU 内存 dump (MB 级别)                      │   │
│  │ │   ├── pages-*.img                                                  │   │
│  │ │   ├── cr-*.img                                                     │   │
│  │ │   └── ...                                                          │   │
│  │ │                                                                     │   │
│  │ └── metadata.json   ← 快照元数据                                     │   │
│  │     {                                                                 │   │
│  │       "volume_id": "sbx-abc123-rootfs",                             │   │
│  │       "juicefs_snapshot": "snap-sbx-abc123-20240108-123456",        │   │
│  │       "created_at": "2024-01-08T12:34:56Z",                         │   │
│  │       "process_config": { ... }                                     │   │
│  │     }                                                               │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  关键点: 文件系统不需要任何操作！                                            │
│          所有写操作已经实时进入 JuiceFS 了                                 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    从快照恢复容器                                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  用户请求:                                                                  │

│  │                                                                      │   │
│  │ Step 2: 挂载快照的 RootFS Volume                                    │   │

│  │                                                                      │   │
│  │ Step 3: CRIU Restore                                                │   │
│  │ └─ criu restore -d -D /snapshots/snap-xyz789/criu                   │   │

│  │                                                                      │   │

│  │     文件系统状态 = 快照时的状态                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  总耗时: ~400ms                                                             │
│                                                                             │
│  关键点:                                                                    │
│  1. 文件系统不需要"恢复" - 它们已经在那里了！                               │
│  2. 只需要恢复内存和进程状态                                               │
│  3. 多个快照可以共享同一个 Volume 的基础数据（CoW）                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

```
│     │   - volume_id                                                   │   │
│     │   - juicefs_snapshot_id                                         │   │
│     │   - criu_dump_s3_path                                           │   │
│     │   - created_at                                                  │   │
│     │   - size_bytes                                                  │   │
│     │   - process_config                                              │   │
│     └─────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 性能对比

### 与传统方案对比

| 指标 | 传统 overlayfs + diff 压缩 | RootFS 直接用 SandboxVolume |
|------|---------------------------|---------------------------|
| **快照创建** | 2-5 秒 | ~600 毫秒 |
| **快照恢复** | 2-5 秒 | ~400 毫秒 |
| **快照大小** | GB 级 (完整 diff) | MB 级 (只有内存) |
| **存储效率** | 每个快照独立 | CoW 共享数据块 |
| **CPU 开销** | 高 (压缩/解压) | 低 (只处理内存) |
| **网络 I/O** | 上传/下载 GB 数据 | 上传 MB 数据 |
| **一致性** | 需要暂停容器 | JuiceFS 事务保证 |

## 注意事项

### 1. 性能考虑

- **FUSE 开销**: 所有文件操作都经过 FUSE，有一定性能开销
  - 本地缓存可缓解 (storage-proxy 默认 1GB 缓存)
  - 对于热文件，缓存命中率很高

- **网络延迟**: JuiceFS 元数据操作需要网络往返
  - 使用连接池
  - 元数据操作批量化

### 2. 兼容性

- **设备文件**: 某些应用可能需要 /dev 下的设备
  - 大部分设备通过 bind mount 挂载
  - FUSE 文件系统不支持创建设备文件

- **Socket 文件**: Unix socket 在 FUSE 上可能有问题
  - 使用 /run 或 /tmp (也在 FUSE 上，但 JuiceFS 支持)
  - 或使用 tmpfs 单独挂载

### 3. 安全性

- **权限隔离**: 确保沙箱之间不能互相访问 Volume
  - JuiceFS 的访问控制
  - Network policies

- **数据隔离**: 不同租户的数据分离
  - 不同 S3 bucket/prefix
  - 不同 PostgreSQL database

## 总结

RootFS 直接使用 SandboxVolume 的方案实现了：

1. **真正的零拷贝快照**: 文件系统无需任何操作
2. **秒级快照/恢复**: 只需处理内存状态
3. **自动持久化**: 任何目录的写操作自动进入 JuiceFS
4. **存储高效**: CoW 共享数据，节省 90%+ 存储
5. **快速冷启动**: 从快照恢复只需 ~500ms

这是相比传统方案的根本性改进，充分利用了 JuiceFS 的分布式文件系统特性。
